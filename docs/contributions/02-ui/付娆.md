## 三、问题与解决
1.  **问题**：状态统计卡片与表单按钮的视觉层级易混淆，用户可能误将统计卡片当作可点击按钮
    **解决**：将统计卡片设计为纯展示型（无 hover/点击效果），按钮添加明显圆角与阴影，通过交互状态区分展示元素与操作元素
2.  **问题**：筛选表单字段较多，易造成页面拥挤
    **解决**：采用 2 列布局排列筛选字段，保持页面整洁，同时保证每个字段的可点击区域足够大，提升移动端适配潜力
3.  **问题**：预约信息表格列数较多，小屏幕下易出现横向滚动
    **解决**：优先展示核心字段（教室名称、时段、日期），次要字段（校区、楼层）可在 hover 时弹窗展示，或在移动端做折叠处理
