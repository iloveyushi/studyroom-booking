### 2. 技术选型

- 前端框架：Vue 3 + Element Plus

  

  理由：Vue 3 具备组合式 API，代码复用性高、响应式逻辑更清晰；Element Plus 提供丰富的表单、表格、弹窗等组件，可快速搭建自习室预约的前端界面，适配 PC 端管理后台与用户端的交互需求，且生态成熟、学习成本低。

- 后端框架：SpringBoot 2.7.x + MyBatis-Plus

  

  理由：SpringBoot 2.7.x 稳定且适配 JDK 11，自动配置特性大幅降低后端开发配置成本；MyBatis-Plus 在 MyBatis 基础上简化单表 CRUD 操作，支持分页、条件查询等预约业务高频场景，同时保留 SQL 自定义能力，适配复杂的预约规则查询需求；整体框架轻量、易扩展，符合中小型项目快速开发的诉求。

- 数据库：MySQL 8.0

  

  理由：MySQL 8.0 开源免费、社区生态完善，支持事务、索引、外键等关系型数据库核心特性，能满足自习室预约系统的 ACID 事务要求（如预约创建时的座位锁定）；8.0 版本性能优化更佳，适配高并发的预约查询场景，且部署成本低、运维简单。

- 部署方式：Docker 容器化部署

  

  理由：Docker 可统一前后端、数据库的运行环境，避免 “开发环境正常、生产环境报错” 的问题；支持快速打包、迁移，便于自习室系统在不同服务器上部署；配合 Docker Compose 可一键启动后端服务与数据库，降低部署复杂度。

## 遇到的问题和解决

1. 问题：后端项目启动时提示 “Cannot resolve symbol'springframework'”，Spring 核心依赖无法识别。

   

   解决：首先检查 pom.xml 配置完整性，替换为适配 JDK 11 的完整依赖配置；清理 Maven 本地缓存，重新执行 clean + install 命令强制下载依赖；配置阿里云 Maven 镜像解决网络下载慢的问题；最终通过 IDEA 重新加载 Maven 项目，成功识别 Spring 核心依赖。

2. 问题：JVM 编译目标版本 25 与关联的 JDK 11 不兼容，导致项目编译失败。

   

   解决：在 pom.xml 中显式指定 java.version 为 11，同步修改 Maven 编译插件的 source 和 target 版本；在 IDEA 中统一项目 SDK、Module 语言级别、编译字节码版本为 11，消除版本冲突。

3. 问题：MySQL 连接时提示 “Access denied”，无法建立数据库连接。

   

   解决：检查数据库用户名 / 密码正确性，确保 MySQL 服务已启动；配置 MySQL 允许远程访问（若部署在不同服务器）；在 application.yml 中添加 useSSL=false、allowPublicKeyRetrieval=true 等参数，解决连接安全校验问题。
