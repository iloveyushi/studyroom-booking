## 遇到的问题和解决
1. 问题：CI 运行目录在 `frontend/src/__tests__`，导致找不到 `jest.config.cjs`。  
   解决：将工作流执行目录统一到 `frontend` 根目录。
2. 问题：测试引用路径写成 `require("./app")`，CI 报 `Cannot find module './app'`。  
   解决：改为 `require("../app")`，并新增 `frontend/src/app.js` 作为桥接。
3. 问题：`git push` 被拒绝（远端先更新）。  
   解决：使用 `git pull --rebase origin develop` 同步后再推送。
4. 问题：`npm ci` 失败（缺 lockfile 或路径不一致）。  
   解决：确保 `frontend/package-lock.json` 在仓库且在 `frontend` 目录执行命令。
