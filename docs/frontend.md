# 前端说明（Frontend Documentation）

## 1. 模块概述

自习室预约系统前端模块主要负责用户界面展示与用户交互，提供用户注册登录、自习室查询、座位预约、预约管理等功能，并通过 HTTP 接口与后端服务进行数据交互。

前端基于 **Thymeleaf + Bootstrap + JavaScript（jQuery）** 技术实现，页面直接集成在 Spring Boot 项目中，用户通过浏览器访问系统即可使用相关功能。

系统前端主要面向两类用户：

- 普通用户（学生）：进行注册登录、座位预约、取消预约等操作
- 管理员用户：进行用户管理、自习室管理、预约管理和数据统计等操作

通过简洁直观的页面设计和响应式布局，系统可以在不同尺寸设备（PC、平板等）上正常显示。

---

# 2. 技术选型

前端模块采用如下技术栈：

| 技术 | 作用 |
|-----|-----|
| Thymeleaf | 模板引擎，用于动态渲染 HTML 页面 |
| Bootstrap | 页面 UI 框架，实现响应式布局 |
| JavaScript | 实现页面交互逻辑 |
| jQuery | 简化 AJAX 请求和 DOM 操作 |
| HTML5 + CSS3 | 页面结构与样式设计 |

技术特点：

- **Thymeleaf** 与 Spring Boot 深度整合，可直接在 HTML 中渲染后端数据。
- **Bootstrap** 提供现成 UI 组件，快速构建美观页面。
- **AJAX 异步请求** 实现前后端数据交互，提高用户体验。

# 3. 前端功能模块

系统前端主要包含以下页面：

## 3.1 登录页面（login.html）

功能：

- 用户输入用户名和密码登录系统
- 提供跳转至注册页面的入口

核心功能：

- 表单输入校验
- 登录请求发送至 `/user/login` 接口
- 登录成功后进入系统首页

---

## 3.2 注册页面（register.html）

功能：

- 新用户注册账号

主要字段：

- 用户名
- 密码
- 确认密码
- 真实姓名
- 手机号码

功能特点：

- 前端校验密码一致性
- 通过 POST 请求调用 `/user/register` 接口完成注册

---

## 3.3 首页（index.html）

功能：

- 展示系统公告
- 显示自习室列表
- 提供预约入口

主要内容：

- 自习室名称
- 自习室位置
- 当前可用座位数量
- 进入预约页面按钮

---

## 3.4 座位预约页面（reserve.html）

该页面是系统核心功能页面。

功能：

- 选择自习室
- 选择预约日期
- 选择开始时间与结束时间
- 查看座位分布图
- 进行座位预约

页面主要区域：

1. **自习室选择区域**
2. **预约时间选择区域**
3. **座位分布显示区域**

座位状态说明：

| 状态 | 显示颜色 |
|----|----|
| 可预约 | 绿色 |
| 已预约 | 灰色 |

用户点击座位按钮即可完成预约操作。

## 3.5 管理员管理页面

管理员端包含以下模块：

### 用户管理

功能：

- 查询用户信息
- 启用 / 禁用用户

### 自习室管理

# 4. 前端模块总结

前端模块采用 Thymeleaf 模板引擎结合 Bootstrap 实现快速开发，页面结构清晰，交互简单易用。

通过 AJAX 与后端接口进行数据交互，实现了用户注册登录、座位预约、预约管理以及管理员管理功能，满足自习室资源预约管理的基本需求。
