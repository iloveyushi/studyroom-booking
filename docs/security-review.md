# 自习室预约系统后端 AI 安全审查报告
审查时间：2026-05-12
审查人：付娆
审查工具：豆包 AI 代码审查

## 一、发现的安全漏洞
### 漏洞 1：SQL 注入风险（高危）
- 位置：MyBatis Mapper XML 文件
- 问题：使用 ${} 直接拼接 SQL，未做参数化处理
- 危害：攻击者可构造恶意 SQL 语句获取/篡改数据库数据

### 漏洞 2：未授权接口访问（中危）
- 位置：部分 Controller 接口
- 问题：未做登录状态校验，游客可直接访问
- 危害：未登录用户可调用敏感接口

### 漏洞 3：敏感信息未脱敏（低危）
- 位置：返回前端的学生信息
- 问题：学号、姓名完整返回，未脱敏
- 危害：用户隐私泄露

## 二、修复方式
1. **SQL 注入修复**
   - 将所有 ${} 改为 #{ }
   - 使用 MyBatis 预编译参数

2. **未授权访问修复**
   - 在 BaseController 增加登录校验
   - 关键接口添加权限判断

3. **敏感信息脱敏修复**
   - 学号只显示前 6 位
   - 姓名只显示姓氏

## 三、修复结果
✅ 所有高危漏洞已修复
✅ 接口权限控制正常
✅ 隐私数据已脱敏
