## 2. 审查结论（前端）

### 发现 1：错误信息直接回显（已修复）
- 漏洞类型：敏感信息暴露（A09 Security Logging and Monitoring Failures / 信息泄露）
- 危害等级：中
- 问题描述：原逻辑将 `error.message` 直接展示到页面，若后端返回 SQL/堆栈细节，可能暴露内部实现。
- 修复措施：新增 `getSafeErrorMessage()`，按场景返回通用错误文案，不再向用户直接透出内部异常细节。
- 修复结果：已在 `frontend/app.js` 中落地。

### 发现 2：登录接口缺少前端限频保护（已修复）
- 漏洞类型：暴力破解风险（认证强化不足）
- 危害等级：中
- 问题描述：登录失败可无限重试，增加撞库/暴力猜解风险。
- 修复措施：前端新增登录失败计数与临时锁定（5 次失败锁定 10 分钟）。
- 修复结果：已在 `frontend/app.js` 中落地（`LOGIN_MAX_ATTEMPTS`、`LOGIN_LOCK_MS`、`loginGuard`）。

### 发现 3：登录输入缺少格式约束（已修复）
- 漏洞类型：输入校验不足
- 危害等级：低
- 问题描述：原始登录输入未做最小长度与字符集约束。
- 修复措施：新增 `validateCredentials()`，限制用户名字符集与长度、密码长度。
- 修复结果：已在 `frontend/app.js` 中落地。
